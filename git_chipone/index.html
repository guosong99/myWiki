总结工作常用的git命令

先配置git user.name 和user.email吧

git config --global user.name "guosong"

git config --global user.email sguo@chiponeic.com

git config --list //查看当前git配置信息

克隆远程仓

git clone ssh://fae@114.242.104.200:7007/code/fingerprint-fae

git clone ssh://fae@114.242.104.200:7007/code/fingerprint-fae

密码：guodaxia

然后

（1）git clone //从远程把代码仓库拿下来

（2）git branch //列出本地仓库下所有分支

​ git branch -a //-a查看本地和远程仓库下的所有分支, -r 仅查看远程分支

​ （fingerprint-fae下的分支： master_xxxxxxx，由AE来维护，可以基于master_xxxxxx来建项目分支）

​ git branch test //创建分支test

​ git checkout test //切换到test分支

​ git push origin test //把分支推到远程分支﻿

​ git branch -d test //删除本地分支test

​ git push origin :haipai_QS5737_msm8937_android8.0_qsee4.0.5//删除远程分支

​ (前面带*号的代表你当前工作目录所处的分支)

（3）例如基于master_android8.0_qsee4这个仓来建立分支

​ git pull //建新项目之前先和远程仓同步到最新

​ git checkout master_android8.0_qsee4 //切换到master_android8.0_qsee4 分支

​ git branch yudeE300_android8.0_qsee4 //基于master_android8.0_qsee4 ，新建一个分支yudeE300_android8.0_qsee4，当前目录下会创建一个product文件夹

​ git checkout yudeE300_android8.0_qsee4 //并切到这个新分支下。在product下修改新分支文件

​ 修改后使用git diff都可以实时查看修改的文件和内容。

​ git status 可以查看修改了哪些文件夹。

（4）提交修改内容

​ git add . //add这个“.”会把仓下所有文件都添加到仓库，包括未跟踪的文件，不建议这么用。

​ git reset //恢复到提交前的状态。

​ git status //可以时刻查看仓库状态（工作区的状态），跟踪所有文件的状态

​ git add product/build_ca.sh //这是比较笨的方法，一个一个提交

​ git add -u //对之前跟踪过的文件，（如果有修改），重新添加到暂存区

​ git commit -m "init ca.sh" //提交到仓库当前分支，并注明提交信息（完成了）【将文件提交到仓库，总共需要两步：add（把文件添加到暂存区）和commit（把暂存区内容提交到当前分支）】

​ git log -2//可以查看最新提交的commit

​ git log -p //查看当前分支的提交log

​ git log --oneline //一行显示所有的提交信息，查看版本

​ git log --pretty=oneline //也是一行显示一条提交信息，但哈希值是完整的。

​ git commit --amend //如果突然又有修改，同样是与上一次提交

一块的，可以使用此命令追加提交

​ git show HEAD//查看每一个commit提交内容

不建议一个项目里只有一个提交，最好多弄几个提交。

（5）撤回

​ reset使用要慎重

​ git reset HEAD~ //软撤回，把上次提交撤回，但改动还在。

​ git reset --hard HEAD~ //硬撤回，所有提交都没有了。

（10）其他常用git命令

​ git diff file.txt //查看文件file.txt之前做了哪些修改，如果工作区有被修改（未提交），具体修改可以用git diff来看，然后再提交。

（11）回退操作（将代码回退到历史某个版本）

git 必须知道当前版本是什么？HEAD表示当前版本（即最新的哪次提交）。HEAD^表示上一个版本，HEAD^^表示上上一个版本，HEAD~100以前的第100个版本。

​ git reset --hard HEAD^ //表示把当前版本回退到上一个版本，--hard的 意思是？

​ git reset --hard （commit id）//--hard 后面跟指定版本的commit id 就可以回到指定版本。

​ git reflog //如果你想回到某个指定版本，版本不记得之前commit id号，可以使用此命令查看历史命令，进而找到历史的commit id

配置gitconfig中git命令行的简写

mac系统下面，如果想为当前用户配置git命令的简写，可以输入如下命令：

cd ~，进入到当前用户目录，这是ls -al 查看一下当前目录下面是否有.gitconfig隐藏文件，如果没有，就touch ~/.gitconfig，新建一个.gitconfig文件出来，如果有就可以直接 vi ~/.gitconfig 直接编辑信息。

编辑内容如下： [alias]

br = branch

保存这个文件，这个时候，就可以 git br 直接查看当前分支，也就是实际的git branch命令。

其他的简写配置可以参考如下：

st = status

ci = commit

至于具体怎么配置git命令的简写，还是要看个人的习惯。

另外，如果想修改其他信息，比如说git操作的用户信息等等 ，都可以在.gitignore配置文件中进行配置
